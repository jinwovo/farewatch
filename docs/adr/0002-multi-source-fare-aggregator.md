# ADR-0002 — 멀티소스 운임 애그리게이터 & 소스 전략

- 상태: Accepted
- 날짜: 2026-06-24

## 맥락
"여러 곳의 가격을 비교해 최저가를 찾아 그 사이트로 보낸다"가 제품의 핵심이다. 자연스러운 질문: 스카이스캐너처럼 **여러 소스를 크롤링/수집**하면 되지 않나?

현실:
- **공개 항공권 가격 API가 사실상 없다.** 스카이스캐너는 파트너 전용(B2B 승인제), 구글 플라이트는 공식 API 없음(QPX Express 2018 종료, 데이터는 구글 소유 ITA 엔진).
- 스카이스캐너/구글의 데이터는 **크롤링이 아니라 계약된 구조화 피드** — GDS(Amadeus/Sabre/Travelport) + 항공사 NDC 다이렉트 + OTA 파트너 피드 + 대규모 캐싱.
- **대부분의 OTA·메타서치는 같은 GDS 재고를 되판다.** → 같은 풀의 사이트를 더 긁어도 가격은 거의 동일. 진짜 가격 분산은 *다른 재고 풀*에서만 나온다: LCC 직판(사우스웨스트·라이언에어 — 메타서치에 아예 없음), 지역 OTA 독점가, 통화/POS(판매국) 차이.
- 정작 그 LCC들이 **스크래핑을 가장 강하게 차단·소송**한다(라이언에어 스크래퍼 상대 판례 다수, 사우스웨스트 직판·봇차단).
- 대규모 스크래핑은 안티봇(DataDome/Akamai) 회피 + 프록시 + 파서 유지보수의 **CI 깨먹는 부채**이고, 공개 레포(실명) ToS 노출이며, 무엇보다 이 프로젝트가 증명하려는 **분산 시스템 근육이 아니다**(봇 회피 근육일 뿐).

## 결정
- 소스를 **`FarePriceProvider` 인터페이스 + 애그리게이터(fan-in → 정규화 → 최저가 채택)** 로 추상화한다(= 스카이스캐너 내부 구조). 소스별 레이트리밋·서킷브레이커·헬스 폴백.
- 커버리지는 **사이트 수가 아니라 *상보적 재고 풀*** 로 확보한다:
  - **Amadeus Self-Service**(무료 테스트티어) — 레거시 항공사(GDS) 백본
  - **Travelpayouts/Aviasales**(제휴, 무료) — 폭넓은 캐시 최저가 + 가격이력. 라이브 예약가가 아닌 점은 "한 달간 최저가 추적"에는 오히려 적합(가격 *신호*만 필요).
  - **LCC 1곳 스크래퍼**(Playwright, 기본 off, 정직성 노트) — GDS가 못 잡는 직판 풀의 *실제* 추가 커버리지 + "스크래핑도 가능" 증명
  - **Simulator** — 합성 가격워크, 부하테스트·상시 데모
- 제품 범위는 **추적 + 알림 + 딥링크**(메타서치). 결제·발권(GDS 발권 API·IATA 인증·PG)은 범위 밖이며 스카이스캐너도 하지 않는다.

## 결과
"많은 데이터"를 *중복이 아니라 상보적으로* 채운다. 엔진(스케줄·레이트리밋·dedup·알림)은 소스 종류와 무관하게 동일하므로, 소스는 가장 안정적·합법적인 조합으로 깔고 엔지니어링 무게는 엔진에 둔다. 스크래퍼는 토이가 아니라 *실제 커버리지를 늘리는* 한 어댑터로만 들어간다.
